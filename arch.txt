================================================================================
                           1. SYSTEM OVERVIEW
================================================================================

The ASTU Digital Lost & Found System is a comprehensive platform designed to
help students at ASTU (Addis Ababa Science and Technology University) report,
search, and recover lost personal belongings.

CORE FEATURES:
- User Authentication (Student/Admin)
- Report Lost Items
- Report Found Items
- Automatic Similarity Matching (80%+ threshold)
- Admin Verification (Both Lost & Found)
- Claim Management
- Trust Score System
- Multiple Notification Channels
- Dashboard Statistics

USER ROLES:
- Student: Can post items, search, claim, respond
- Admin: Can verify items, approve/reject claims, view stats

================================================================================
                    6. SIMILARITY MATCHING ENGINE
================================================================================

TRIGGER: When new item is verified (lost or found)

COMPARISON LOGIC:
- If new item is LOST: Compare with ALL "found" items
- If new item is FOUND: Compare with ALL "lost" items

SIMILARITY CALCULATION:

┌─────────────────┬──────────┬───────────────────────┐
│ Factor          │ Weight   │ How Calculated        │
├─────────────────┼──────────┼───────────────────────┤
│ Title           │  30%    │ Levenshtein distance │
│ Similarity      │          │ or word match        │
├─────────────────┼──────────┼───────────────────────┤
│ Category        │  30%    │ Exact = 100%         │
│ Match           │          │ Partial = 50%        │
├─────────────────┼──────────┼───────────────────────┤
│ Description     │  20%    │ Word overlap ratio   │
│ Similarity      │          │                      │
├─────────────────┼──────────┼───────────────────────┤
│ Location        │  10%    │ Same building = 100% │
│ Proximity       │          │ Same campus = 50%    │
├─────────────────┼──────────┼───────────────────────┤
│ Date            │  10%    │ Same day = 100%      │
│ Proximity       │          │ ±3 days = 70%        │
└─────────────────┴──────────┴───────────────────────┘

FORMULA:
TOTAL = (Title × 0.3) + (Category × 0.3) + (Description × 0.2) + 
        (Location × 0.1) + (Date × 0.1)

THRESHOLD: If TOTAL ≥ 80% → MATCH!

FOR EACH MATCH:
- Save similarity_log
- Mark item as "has_match"
- Send notification to LOST item owner

================================================================================
                    9. OWNER RESPONSE & CLAIM FLOW
================================================================================

IF OWNER RESPONDS "YES, IT'S MINE!":

1. Owner fills claim form:
   • Proof of ownership (description, receipt, etc)
   • Additional notes

2. Save claim:
   • item_id: matched found item
   • user_id: owner
   • similarity_score: (from match)
   • status: "pending"
   • proof: [owner's submission]

3. Notify:
   • Admin: "New claim submitted"
   • Finder: "Owner has claimed your item"

IF OWNER RESPONDS "NO, NOT MINE":
- Item stays "available"
- System continues searching
- Other owners can also get notification

================================================================================
                      15. SECURITY REQUIREMENTS
================================================================================

1. SECURE FILE UPLOAD
   • Rename files (timestamp_random.extension)
   • Validate MIME type (not just extension)
   • Store outside web root or use storage
   • Limit file size (max 2MB)

2. ROLE-BASED ACCESS CONTROL
   • Middleware: 'auth' for all routes
   • Middleware: 'admin' for admin routes
   • Check: if (auth()->user()->role === 'admin')

3. SQL INJECTION PREVENTION
   • Use Eloquent ORM (automatic escaping)
   • Use parameterized queries
   • Input validation

4. PASSWORD SECURITY
   • Hash with bcrypt (Laravel Hash::make)
   • Never store plain text

5. INPUT VALIDATION
   • Laravel Form Requests
   • Required fields
   • Max lengths
   • Email format
   • File type validation

6. PROTECT USER DATA
   • Auth check before showing user data
   • {{ auth()->id() === $item->user_id }}
   • No direct user info in URLs

7. PREVENT FALSE CLAIMS
   • Trust score system
   • Similarity matching before claim
   • Admin verification
   • Claim limits

================================================================================
                        16. TRUST SCORE SYSTEM
================================================================================

START: Everyone starts at 0

GOOD ACTIONS (+ points):
• +1 for verified lost item posted
• +1 for item returned to owner
• +1 for verified found item posted

BAD ACTIONS (- points):
• -1 for rejected item (fake/spam)
• -3 for false claim
• -3 for reported behavior
• -5 for multiple false claims

TRUST LEVELS:
• Score above +5: "Trusted User" - Fast approval
• Score 0 to +5: "Normal User" - Normal process
• Score below 0: "Warning" - Extra verification
• Score below -10: "Blocked" - Cannot post items
